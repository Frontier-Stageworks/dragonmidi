# Application Shell, Signal Monitor & Status UI — EARS Specs

Traces to `docs/llds/app-ui.md`.

## Signal Monitor

- [x] **UI-MONITOR-001**: The system shall initialize each channel's (`midi`, `dragonframe`) last-activity value to a "never seen" sentinel, distinct from any real timestamp, at startup.
- [x] **UI-MONITOR-002**: The system shall compute liveness using a monotonic clock, such that a channel is reported live only while its last-activity value is not the "never seen" sentinel and the elapsed monotonic time since that value is strictly less than the liveness window (proposed 2.0s).
- [x] **UI-MONITOR-003**: While a channel's error flag is set (MIDI: Native Mode handshake failure per `MIDI-NATIVE-003`, only possible under a profile with `has_native_mode` true; Dragonframe: listener bind failure per `OSC-LISTEN-003`), the system shall report that channel's display state as error, taking precedence over both live and quiet.

## Controller Profile Selector

- [x] **UI-PROFILE-001**: The system shall display a Controller Profile dropdown populated from the discovered, merged Controller Profile list (`PROFILE-LOAD-001`/`003`/`004`, see `docs/specs/midi-input.md`), defaulting the active selection to the profile named "nanoKONTROL Studio" on launch (`MIDI-PROFILE-004`) regardless of that list's display order.
- [x] **UI-PROFILE-002**: When the user changes the Controller Profile dropdown, the system shall apply the change immediately, with no separate Apply step (`MIDI-PROFILE-005`/`006`, see `docs/specs/midi-input.md`).
- [x] **UI-PROFILE-003**: While the active Controller Profile's `setup_hint` is present and non-empty (`PROFILE-LOAD-009`, see `docs/specs/midi-input.md`), the system shall display it verbatim as a one-line hint beneath the dropdown; while it is absent or empty, no hint line shall be shown.
- [x] **UI-PROFILE-004**: While the config-file failure count (`PROFILE-LOAD-011`, see `docs/specs/midi-input.md`) is greater than zero, the system shall display a one-line, count-only indicator near the Controller Profile dropdown (e.g. "2 controller config files failed to load"), independent of and able to be shown simultaneously with the `UI-PROFILE-003` setup hint; while the count is zero, no such indicator shall be shown. This indicator shall not display individual file names or error messages in the main window.

## Status UI

- [x] **UI-STATUS-001**: The system shall display two indicator rows, "MIDI signal" and "Dragonframe signal," each rendering a 3-state dot (live / error / quiet) and a short label, both derived from a single Signal Monitor read per UI update tick.
- [x] **UI-STATUS-002**: While the MIDI channel has not yet connected to a device matching the active Controller Profile (per `MIDI-CONN-007`, see `docs/specs/midi-input.md`), the system shall display "Waiting for {active profile's name}…" as the MIDI row's secondary text; when connected, it shall display the connected device's name instead.
- [x] **UI-STATUS-003**: The system shall display the configured local listen port on the Dragonframe row, and the configured Dragonframe host and port in an editable "Sending to" field pair.
- [x] **UI-STATUS-004**: The system shall treat the indicator dot's live/error/quiet state and the row's secondary label (connection/device-name text) as independent signals; a channel may simultaneously display an error dot and a connected-device label (e.g., a physically connected controller whose Native Mode handshake failed), and this combination is intended, not a defect. Under the nanoKONTROL2 profile, the error dot never lights (`MIDI-NATIVE-005`), so this combination cannot occur for that profile.
- [x] **UI-CONFIG-001**: When the user edits the host, Dragonframe port, or listen port fields and presses Apply, the system shall validate and apply the new configuration; edits shall not take effect, and no rebind shall occur, before Apply is pressed. Applying a changed local listen port shall trigger the OSC Listener to close and rebind its socket (`OSC-LISTEN-006`, see `docs/specs/osc-io.md`).
- [D] **UI-CONFIG-002**: Where settings persistence is enabled, the system shall persist the Dragonframe host, Dragonframe port, local listen port, and the selected Controller Profile across app launches (deferred pending a decision on whether phase 1 includes any persisted state).

## Mapping View

**As of 2026-07-23, the rendered table has 3 columns — Name, MIDI, Target — not 5.** Trigger and Target type are no longer displayed as their own columns; `build_rows()`/`RowView` (`dragonmidi/mapping_view_model.py`) still compute both fields (referenced below where they name what *kind* of target a row has), but nothing in the Mapping View renders them on screen. See `UI-MAP-018`.

- [x] **UI-MAP-001**: The system shall render one table row per entry in the active Controller Profile's opinionated map (`MIDI-PROFILE-002`/`003`, see `docs/specs/midi-input.md`), in table order, showing that entry's name, MIDI source, and target — except Knob and Mute entries, which shall not be rendered as their own rows; their behavior (`MAP-BANK-001`, `MAP-BANK-002`, `MAP-BANK-004`, see `docs/specs/static-mapping.md`) is unaffected, only their table display is folded into their bank's fader row. Stop, Cycle, and Previous/Next Marker are ordinary (non-folded) entries under this rule, each rendering its own row with its WebSocket target (`MAP-WS-001`, `MAP-WS-003`/`004`, `MAP-WS-006`/`007`, see `docs/specs/static-mapping.md`); Solo is not rendered under this rule at all — see `UI-MAP-013`. Previous Track and Next Track are likewise not rendered under this rule as of Phase 6, since they are no longer opinionated-map entries (`MAP-GROUP-003`, see `docs/specs/static-mapping.md`) — see `UI-MAP-016`. Because the nanoKONTROL2's opinionated map omits the Scene button entirely (`MIDI-PROFILE-003`), no Scene row is rendered under that profile — not a disabled placeholder, simply absent, since it is not in that map to iterate over.
- [x] **UI-MAP-002**: The system shall permit editing the target only on the 8 fader rows — the Group axis-name/min/max values within the Target cell (`MAP-AXIS-004`, see `docs/specs/static-mapping.md`); non-fader, non-bank-member rows (transport/marker buttons, Previous/Next Track, and the Scene button under a profile that has one) shall display their target with no control to change it. As of `UI-MAP-018`, no row offers a target-*type* choice; a fader's OSC axis/OSC encoder mode is not user-selectable from this view.
- [x] **UI-MAP-018**: The Mapping View shall render exactly 3 columns — Name, MIDI, Target — omitting Trigger and Target type entirely (removed 2026-07-23; `UI-MAP-003`, which described the now-removed OSC encoder/OSC axis toggle UI, is deleted, not deprecated). Every fader row shall behave as if permanently in OSC axis mode from this view — its 5 Group pickers (`UI-MAP-014`) always visible — regardless of `MappingEngine.is_axis_mode`'s actual value; `MappingEngine.clear_axis_target`/`enter_axis_mode` remain part of the engine's public interface but are not called from anywhere in this view. The system shall additionally render a header row of 5 column letters, "A" through "E" left-to-right, as an actual row of the same table (not a separate widget) so its column boundaries match every fader row's Group picker columns.
- [x] **UI-MAP-004**: For each of a fader row's 5 Group pickers, the system shall recompute its candidate list from `AxisDiscovery.axes` on every UI update tick (the same tick that updates the status indicators), not only when the picker is opened or Rescan is pressed, and shall not accept a free-text or arbitrary name (`MAP-AXIS-003`, see `docs/specs/static-mapping.md`); all 5 pickers on a row share the same candidate list, since axis discovery is project-wide, not per-Group.
- [x] **UI-MAP-005**: While `AxisDiscovery.axes` is `None` (never queried, or a query outstanding), the system shall render every Group picker's candidate list as disabled with the text "Discovering…"; while `AxisDiscovery.axes` is an empty dict (queried, zero axes found), the system shall render every Group picker as disabled with the text "No axes found"; otherwise the system shall render every Group picker enabled with the discovered names, sorted for stable display order.
- [x] **UI-MAP-006**: When the user selects an axis name or edits a min/max field on one Group's picker within a fader row set to OSC axis, the system shall call `MappingEngine.set_axis_target(key, group, name, min, max)` — `group` identifying which of the row's 5 pickers was edited — once both a name is selected and both fields parse as real numbers, with no separate save step and with no requirement that `group` equal the currently active Group (`MAP-GROUP-008`); until then, that Group's assignment continues driving whatever target it was last actually given (`MAP-AXIS-006`).
- [x] **UI-MAP-007**: The system shall accept any value parseable as a real number in any Group picker's min/max fields, including `min > max` or `min == max` (`MAP-AXIS-002`), and shall not reject or warn on either case; text that fails to parse as a number shall not be applied, and that picker's last successfully-applied target (if any) shall remain in effect, with no error dialog.
- [x] **UI-MAP-008**: If a fader row's Group picker's configured axis name is no longer present in `AxisDiscovery.axes`, the system shall continue showing that name as that picker's current selection (greyed to indicate it cannot be re-selected once deselected), rather than clearing or replacing it, and shall not alter that Group's configured target as a result (`MAP-AXIS-006`, see `docs/specs/static-mapping.md`). This is evaluated independently per Group picker — one Group's assignment going stale does not affect any other Group's picker on the same row.
- [x] **UI-MAP-009**: When the user activates "Rescan axes," the system shall call `OscListener.rescan()` and shall not modify any row's already-configured target, in any Group, as a result of the call itself.
- [x] **UI-MAP-010**: As of Phase 6, the (Bank, Group) axis-name/min/max table persists across application launches via the Preset Store (`MAP-STORE-001`ff., see `docs/specs/static-mapping.md`); every other piece of Mapping View state — each fader's OSC axis/OSC encoder toggle, and every non-fader row's content — holds no persisted state and resets to its default on next launch, matching this spec's pre-Phase-6 behavior for everything except the per-Group axis table.
- [x] **UI-MAP-011**: On initial render, the system shall show each fader row with all 5 Group pickers visible; a Group picker whose (Bank, Group) pairing has a value loaded from the Preset Store (`MAP-STORE-002`) shall show that value pre-selected, and a Group picker with no persisted value shall show no name selected (`MAP-AXIS-008`, see `docs/specs/static-mapping.md`).
- [x] **UI-MAP-012**: While the active Controller Profile's `has_jog_wheel` is true (the nanoKONTROL Studio), in addition to the opinionated-map-derived rows (`UI-MAP-001`), the system shall render two further rows for the jog wheel, since it is not itself an opinionated-map entry (`docs/specs/static-mapping.md § Jog Wheel Frame Stepping`): a row named "Jog Wheel" with Target type "OSC action" and Target "stepForward / stepBackward", and a row named "Jog Wheel (Arc)" with Target type "Keystroke" and Target "Option+Shift+Right / Option+Shift+Left"; both rows shall show MIDI source "CC110, ch{profile's default channel}" and trigger "Directional", and neither shall be editable. While `has_jog_wheel` is false (the nanoKONTROL2), the system shall render neither row.
- [x] **UI-MAP-013**: The system shall render exactly one row named "Solo 1-8" for the Solo 1–8 controls, not folded into any fader row (unlike Knob/Mute, `UI-MAP-001`) and not rendered as eight separate rows; the row shall show MIDI source "CC32-39, ch{active profile's default channel}", trigger "Press", Target type "WebSocket", and shall not be editable. This row is rendered identically under both Controller Profiles, since both have Solo 1–8. As of Phase 6, the row's Target text shall be recomputed every UI tick from the active Group `g` (`MAP-GROUP-002`) as "select-AX{1+8(g-1)} – select-AX{8+8(g-1)} (Group {g})" (`MAP-WS-002`, see `docs/specs/static-mapping.md`) — e.g. "select-AX9 – select-AX16 (Group 2)" — showing the live axis range rather than the pre-Phase-6 static "select-AX1 – select-AX8 (button N → AXN)" text. The "button N → AXN" phrasing is dropped in favor of the plain Group number, avoiding an awkward "+0" offset term at Group 1.
- [x] **UI-MAP-014**: Each of the 8 fader rows shall show 5 axis-name pickers, one per Group, ordered left-to-right as Group 1 through Group 5, replacing the single picker of the pre-Phase-6 design; each picker independently applies `UI-MAP-004` through `UI-MAP-008`.
- [x] **UI-MAP-015**: The system shall render a Group indicator row, positioned above the 8 fader rows within the Target column's space, showing 5 lights ordered left-to-right as Group 1 through Group 5; the light corresponding to the currently active Group (`MAP-GROUP-002`, see `docs/specs/static-mapping.md`) shall render blue, and every other light grey; this row shall be recomputed every UI tick, the same cadence as `UI-MAP-004`'s picker candidates and `UI-MAP-013`'s Solo row text.
- [x] **UI-MAP-016**: The system shall render two additional fixed rows, "Previous Track" and "Next Track," the same way `UI-MAP-012`'s jog-wheel rows and `UI-MAP-013`'s Solo row are rendered outside the opinionated-map iteration (`UI-MAP-001`). Each row's Target type shall be "Group switch" (a sixth read-only Target type value, alongside OSC action/encoder/axis/Keystroke/WebSocket); Target text shall read "Previous (wraps 5→1)" / "Next (wraps 5→1)" respectively (`MAP-GROUP-004`, see `docs/specs/static-mapping.md`). Neither row shall be editable. Where a profile declares only one of `previous_track`/`next_track` (`MAP-GROUP-006`), the system shall render only the corresponding row.
- [x] **UI-MAP-017**: When the user clears one Group picker's selection back to "no axis chosen" (rather than toggling the row's OSC encoder/OSC axis Target type), the system shall call `MappingEngine.clear_group_axis_target(key, group)` (`MAP-GROUP-009`, see `docs/specs/static-mapping.md`), not `clear_axis_target(key)` — leaving the fader in OSC axis (direct) mode and every other Group's assignment for that fader unaffected.

## Threading and Shutdown

- [x] **UI-THREAD-001**: The system shall bridge MIDI events and OSC Listener datagrams from their background threads into the Qt main thread using thread-safe queues drained by a timer, fully draining each queue on every tick with no per-tick cap.
- [x] **UI-SHUTDOWN-001**: When the application quits, the system shall run each cleanup step (Native Mode release, MIDI port close, OSC Listener stop, UDP client socket close) with its own isolated error handling, such that a failure in one step does not prevent the others from running, and shall bound the overall shutdown sequence with a timeout before force-exiting.
